# SOURCE_POLICY

## Principle
External signals are candidates for attention, not truth.

## Intake model
For each signal, capture and keep separate:
- confirmed_facts: Verifiable and corroborated information.
- assumptions: Working hypotheses used for prioritization.
- unverified: Claims not yet validated.
- risks: Potential downside if interpretation is wrong.

## Source handling
- Prefer transparent, attributable source URLs.
- Keep registry entries stable via unique IDs.
- Record source type, cadence, and reliability hints.

## Prohibited in this skeleton
- Automated scraping implementations.
- Secret/API key handling.
